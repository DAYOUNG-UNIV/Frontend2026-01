Step 3. HTML Form 복습 (1/2)

📝 HTML Form 이란?

사용자가 웹 사이트로 정보를 보낼 수 있게 해주는 핵심 요소입니다.
오늘은 이를 활용해 상품 주문 페이지를 만들어 봅니다.

<input type="text">

글자 입력칸 (이름, 요청사항 등)

<input type="radio">

단일 선택 (사이즈, 온도 등)

<input type="checkbox">

다중 선택 (추가 토핑 등)

<select>

드롭다운 메뉴 (수령 시간 등)

Step 3. 패스트오더 기획 회의 (2/2)

조별 회의 10분

"학교 근처 맛집 패스트오더 만들기"

조별로 명지전문대 근처의 맛집(음식점 혹은 카페)을 하나 선정하고,
해당 가게의 메뉴를 주문할 수 있는 폼(Form)의 구성을 기획합니다.

식당 선정 기준: '도보'로 갈 수 있는 곳

※ 교수님 피셜: "저는 퇴근할 때 2시간 걸어서 집에 간 적도 있습니다. 본인 체력만 허락한다면 2시간 거리 맛집도 '학교 근처'로 흔쾌히 인정해 드립니다. 🏃‍♂️💨"

Step 4. 종이 위에 UI 스케치 ✏️ (1/2)

회의한 내용을 바탕으로 실제 주문 화면을 스케치합니다. (15분 진행)

필수 제약 조건

그냥 빈 네모칸만 그리지 마세요. 오늘 배운 Form 태그들이 화면에 어떻게 배치될지 구체적으로 스케치해야 합니다.
다음 4가지 요소 중 최소 3가지 이상을 반드시 포함하세요.

  • 단답/장문 텍스트 (요청사항 등)
  • 라디오 버튼 (사이즈 M/L 택1 등)
  • 체크박스 (추가 토핑 다중선택 등)
  • 셀렉트 박스 (수령 시간 선택 등)
