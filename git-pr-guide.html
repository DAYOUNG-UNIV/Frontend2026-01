Intro

모두 함께 PR 해보기

이번 팀 과제의 가장 중요한 목표는 '모든 팀원이 PR이 무엇인지 이해하고 직접 해보는 것'입니다.

코드를 작성하기 전에 '글(Markdown)'로 된 기획서(README.md)를 다 함께 완성해 보면서 깃허브 협업 흐름을 경험해 보겠습니다.

필수사항 및 제출 방법

[필수사항 및 제출 방법]
  • 제출자: 팀원 중 대표 1명만 제출 (반드시 팀장일 필요 없음, 상의하여 등록)
  • 제출물: 팀원 전원의 PR 내역이 병합(Merge)되어 기획서가 완성된 팀 깃허브 레포지토리 링크
  • 교수진이 해당 링크에 들어가 모든 팀원이 PR을 생성했는지 기록을 확인합니다.

참고 사항

처음 보는 에러에 당황하는 것은 아주 당연합니다! 막히면 혼자 고민하지 말고 화면을 캡처해서 팀원들에게 SOS를 치거나, 디스코드 질의응답 방에 질문해 주세요.

회의 시간과 일정은 Scrum Master가 책임지고 잘 이끌어 주시기 바랍니다.

팀 회의 안건 (Step 1)

  • 우리 서비스의 이름
  • 타겟 유저 (페르소나)
  • 핵심 기능 3가지 (3주 내 구현 가능성 고려)
  • 역할 분담: 기획서 템플릿의 어느 부분을 누가 타이핑할지 정확히 나누기
