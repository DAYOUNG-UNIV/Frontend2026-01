Step 4-1. 티어 리스트 레이아웃 분석

작성해야 할 TODO 영역의 Flexbox 적용 원리를 분석합니다.

가로 배치 (라벨과 아이템 영역)

.tier-row {
  display: flex;
}

각 등급을 나타내는 행(Row)입니다. 기본값인 flex-direction: row;를 적용하여 좌측의 라벨(S, A..)과 우측의 아이템 드롭 영역이 가로로 나란히 배치되도록 합니다.

유연한 줄바꿈 (Flex-wrap)

.tier-items {
  display: flex;
  flex-wrap: wrap;
}

특정 등급에 많은 이미지가 몰릴 경우, 요소들이 영역을 벗어나지 않고 자연스럽게 다음 줄로 배치되도록 설정하는 속성입니다.

잔여 공간 채우기 (Flex-grow)

.tier-items {
  flex-grow: 1;
}

고정된 너비를 가진 요소(예: 등급 라벨, 사이드 패널)를 제외하고, 남은 가로 공간을 특정 영역(드롭 영역 또는 메인 보드)이 꽉 채우도록 점유율을 지정합니다.

반응형 화면 대응 (Media Query)

@media (max-width: 800px) {
  .tier-row { flex-direction: column; }
}

디바이스 화면 너비가 특정 크기 이하로 줄어들면, 기존에 가로로 배치되던 영역들을 세로(column) 방향으로 전환하여 모바일 환경에서도 가독성을 유지합니다.

수업 시간에 완성한 조별 티어 리스트 코드를 바탕으로, 항목들을 나만의 기준에 따라 S~D 등급으로 재배치합니다. Flexbox 속성을 세밀하게 조정하여 가장 완성도 높은 웹페이지를 제출하세요.

레이아웃 필수 적용 요소

  • 나만의 배치
    조별 실습 때 합의했던 순서는 잊으세요! 오직 내 취향과 평가 기준에 맞춰 아이템들의 티어를 새롭게 지정해 봅니다. HTML 코드의 위치를 직접 다른 등급으로 옮기거나, CSS의 order 속성을 활용하여 화면에 출력되는 순서를 손쉽게 바꿀 수 있습니다.
  • 정렬 최적화
    justify-content, align-items, gap 등을 조절하여 각 등급 내 아이템들이 가장 깔끔하게 정렬되도록 미세 조정
  • 반응형 유지
    모바일 화면 크기로 줄였을 때 뷰포트가 깨지지 않고 자연스럽게 세로 레이아웃으로 변경되는지 확인

과제 제출 형식

  • 필수 스크린샷 2장:
    1. 나만의 기준으로 재배열이 완료된 PC 전체 화면 캡처
    2. 브라우저 폭을 줄여 모바일 레이아웃으로 변경된 화면 캡처
  • 정렬을 위해 추가/수정한 주요 CSS 코드 블록 (마크다운 포맷)
  • GitHub Pages 배포 링크 필수 포함
  • 마크다운 작성 후 PDF 문서로 변환하여 제출
